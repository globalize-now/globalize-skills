### Deviations from SKILL.md
- **`<Select>` macro prop naming.** The setup/convert SKILL shows `<Select value={gender} male="..." female="..." other="..." />`, but the installed `@lingui/react@5.9.5` types require an underscore prefix (`_male`, `_female`) for named branches — bare `male`/`female` fails TS with "Property 'male' does not exist on type 'SelectChoiceProps'". Same applies to `<Plural>` custom cases (though standard `one`/`other` are fine). Skill examples need updating to the `_branch` syntax.
- **Dynamic catalog import pattern.** The skill says to use string concatenation (`"./locales/…" + params.locale + ".ts"`) to defeat esbuild's analyzer, but the experimental extractor's esbuild still tries to resolve that concatenation at extract time. Extract fails with "Could not resolve import" if the target `.ts` files don't already exist. Workaround: seed empty `.po` files and run `lingui compile --typescript` once before the first `lingui extract-experimental`. The skill's "Scaffold and Verify" step runs extract before compile, which breaks on a fresh project with per-route dynamic imports. Worth documenting the ordering or suggesting an initial compile-of-empties.
- **TanStack Router plugin file-routing collision.** When per-page catalogs live at `src/routes/$locale/locales/{page}/{locale}.ts`, the TanStack router plugin treats them as route files and warns (or registers them as routes). Had to add `routeFileIgnorePattern: "locales/"` to `tanstackRouter()`. The skill's TanStack Router guidance in `vite-babel.md` doesn't mention this; newcomers will hit `routeTree.gen.ts` polluted with fake routes.
- **TanStack `redirect({ to: ... })` typing.** The skill's Strategy 2 example uses `redirect({ to: \`/${SOURCE_LOCALE}${location.pathname}\` })`. TanStack Router's `to` is strictly typed to known routes, so that template literal fails TS. Had to switch to `redirect({ href: ... })`. Worth amending the reference.
- **`useParams` for non-strict routes.** The `useParams({ from: "/$locale/" })` pattern was used for typed access inside `HomePage` to get `locale` for the `<Link>` inside `<Trans>`. Worked, but skill doesn't show this mixed usage explicitly.

### Overall verdict
A real user following unguided mode would land in a compiling, buildable project — `npm run build` succeeds, 12 messages extract correctly with proper ICU plural/select/interpolation and comments, the language switcher is wired, redirects for bare paths work, per-page catalogs are colocated. They would, however, hit three speed bumps that the skill should fix: (1) `<Select>`/`<Plural>` named-case syntax discrepancy with current types, (2) the extract-before-compile chicken-and-egg on fresh projects using per-route dynamic imports, and (3) TanStack Router plugin conflating catalog files with route files without a documented `routeFileIgnorePattern`. Weakest link is the per-page catalog flow on TanStack Router — accurate advice exists for the scaffolding shape, but the bootstrap sequencing and router-plugin interaction aren't covered. The convert skill is solid: macro decision tree produced the right wraps, including a clean rewrite of the `itemsLabel` if-chain into a single `<Plural>` and the gender ternary into `<Select>`, plus `i18n.number`/`i18n.date` replacing the hardcoded `en-US` formatters.
